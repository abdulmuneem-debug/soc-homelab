02 · Setup Process

How I Built It

Step-by-step process to replicate this environment from scratch.

01
Install VirtualBox and Create VMs
Downloaded Oracle VirtualBox 7.x on Windows. Created two VMs: Kali Linux (80GB dynamic VDI, 2GB RAM, NAT network) and Windows 10. Took a saved snapshot of Kali after base configuration so I can roll back anytime.
02
Configure Kali Linux Analyst Environment
Installed Wireshark, updated package lists, and verified sudo access. Used sudo su for root sessions when needed. Set up Firefox with VirusTotal and CyberDefenders bookmarks for CTF work.
03
Install Splunk Enterprise on Windows VM
Installed Splunk Enterprise on the Windows 10 VM. Configured it to ingest Windows Event Logs (System, Security, Application) and CBS logs from C:\Windows\Logs\CBS\. Splunk accessible at localhost:8000.
04
Verify Log Ingestion in Splunk
Ran index=main to confirm data flow. Verified 111,228+ events were indexed correctly with proper source, host, and timestamp fields. Confirmed TrustedInstaller and system log events were parsing as expected.
05
Practice on CyberDefenders Blue Team CTFs
Loaded challenge PCAPs (like WebStrike) into Wireshark on the Kali VM. Practiced following HTTP streams, extracting IOCs, identifying attack techniques, and correlating findings with VirusTotal threat intelligence.

SS-04 CyberDefenders CTF · WebStrike · Wireshark
Detected: LFI Attack via HTTP POST
In the WebStrike challenge on CyberDefenders, I analyzed a PCAP file to identify an active web attack. Using Wireshark's "Follow HTTP Stream" on TCP stream 14, I identified a Local File Inclusion attack. The attacker sent a POST request using curl/7.81.0 with /etc/passwd as the payload body — a classic LFI attempt to read sensitive Linux system files.
# HTTP Stream captured from tcp.stream eq 14

POST / HTTP/1.1
Host: 117.11.88.124:443
User-Agent: curl/7.81.0
Accept: */*
Content-Length: 11
Content-Type: application/x-www-form-urlencoded

/etc/passwd   ← LFI payload
CRITICAL Local File Inclusion (LFI) — /etc/passwd Exfiltration Attempt
Source IP: 24.49.63.79  →  Destination: 117.11.88.124:443
Method: POST  ·  Tool: curl/7.81.0 (automated)
ATT&CK Technique: T1083 — File and Directory Discovery / LFI
The attacker leveraged a vulnerable web form to attempt path traversal and read /etc/passwd, which would reveal system usernames. Curl's User-Agent suggests a scripted/automated attack.
HIGH Non-Browser User-Agent on HTTPS Port
Traffic from curl on port 443 is a red flag in web server logs. Legitimate users rarely send curl requests to production web applications. This is a strong indicator of automated exploitation tooling.

The hash belongs to a PE DLL file (labeled PEDLL on VirusTotal), which indicates it likely functions as a dropper, loader, or injected module. The graph shows it communicating with multiple PE executable files — suggesting it drops or fetches secondary payloads. This is typical of staged malware like RATs or stealers.
MEDIUM Graph-Based IOC Expansion Technique Used
By pivoting from the initial hash to related nodes on VirusTotal Graph, additional IOCs (IPs, domains, child hashes) can be extracted for threat hunting rules in Splunk or firewall blocklists. This is a standard threat intelligence enrichment workflow.
